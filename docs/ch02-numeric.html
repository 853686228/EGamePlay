Chapter 02

FloatNumeric — 数值系统

战斗数值的一切基础。HP、攻击力、防御、暴击率、移速...所有可变属性都通过 FloatNumeric 管理。它的核心设计目标：所有修改可逆 — Buff 结束时移除 Modifier，数值精确恢复。

2.1 概述

核心问题
一个角色的攻击力 = 基础值 + 装备加成 + Buff 加成 + 被动技能 + ...。当某个 Buff 结束时，如何精确回退它的效果？

错误做法： atk = atk - buffValue — 浮点误差累积，且百分比加成无法简单减回来
正确做法： 每个修改源持有自己的 Modifier 引用，移除时从集合中删除，重新计算最终值
文件位置
目录
Assets/App.Model/Game.Model/.../CombatEntity/Attribute/
├── FloatNumeric.cs          // 带修改器的数值（69行）
├── FloatModifier.cs         // 修改器和修改器集合（44行）
└── [AttributeComponent.cs]  // 在 CombatEntity 目录下（29行）

2.2 FloatModifier — 修改器

FloatModifier 是最小单元 — 一个可变的浮点数值。每次 Buff 给角色加攻击力时，就创建一个 FloatModifier 并添加到对应集合中。

FloatModifier.cs
namespace EGamePlay.Combat
{
    /// 单个修改器 — 代表一个修改源的贡献值
    public class FloatModifier
    {
        public float Value;  // 修改值（如 +20 攻击力，或 +50% 百分比）
    }

    /// 修改器类型枚举
    public enum ModifierType
    {
        Add,          // 基础加法（第一层）
        PctAdd,       // 百分比加法（第二层）
        FinalAdd,     // 最终加法（第三层）
        FinalPctAdd,  // 最终百分比（第四层）
    }
}
为什么 Modifier 是 class 不是 struct
因为需要引用语义。当 Buff 结束时，我们需要找到并移除「那个特定的 Modifier 实例」，而不是通过值匹配。两个 Buff 可能都加 20 攻击力，但移除其中一个不能影响另一个。

2.3 FloatModifierCollection — 修改器集合

多个 Modifier 组成一个 Collection（如所有加法修改器放一个集合）。集合负责自动维护 TotalValue（所有 Modifier.Value 之和）。

FloatModifier.cs（续）
public class FloatModifierCollection
{
    /// 当 TotalValue 变化时通知外层 FloatNumeric 重算
    public event Action UpdateEvent;

    /// 所有修改器的值之和
    public float TotalValue { get; private set; }

    /// 内部存储
    private List<FloatModifier> _modifiers = new();

    /// 添加修改器
    public void AddModifier(FloatModifier modifier)
    {
        _modifiers.Add(modifier);
        TotalValue += modifier.Value;  // 增量更新，O(1)
        UpdateEvent?.Invoke();          // 通知 FloatNumeric 重算最终值
    }

    /// 移除修改器
    public void RemoveModifier(FloatModifier modifier)
    {
        _modifiers.Remove(modifier);
        TotalValue -= modifier.Value;  // 增量更新，O(1)
        UpdateEvent?.Invoke();
    }

    /// 全量重算（用于修改器值被外部改变后）
    public void Update()
    {
        TotalValue = 0;
        foreach (var m in _modifiers)
            TotalValue += m.Value;
        UpdateEvent?.Invoke();
    }
}
增量更新 vs 全量重算
Add/Remove 时直接 ±= 单个值，O(1) 时间复杂度。只有在 Modifier.Value 被外部修改（罕见情况）时才需要全量 Update()。这对于频繁 Buff 增删的 MOBA 场景非常关键。

2.4 FloatNumeric — 完整数值类

FloatNumeric 持有一个 baseValue 和四个 ModifierCollection，任何一个变化都会触发重算最终 Value。

FloatNumeric.cs — 完整源码
namespace EGamePlay.Combat
{
    public class FloatNumeric
    {
        // ═══════════ 最终计算结果 ═══════════
        public float Value { get; private set; }

        // ═══════════ 基础值 ═══════════
        public float baseValue;

        // ═══════════ 四层修改器集合 ═══════════
        public FloatModifierCollection add          = new();  // 加法层
        public FloatModifierCollection pctAdd       = new();  // 百分比层
        public FloatModifierCollection finalAdd     = new();  // 最终加法层
        public FloatModifierCollection finalPctAdd  = new();  // 最终百分比层

        // ═══════════ 类型映射 ═══════════
        public Dictionary<ModifierType, FloatModifierCollection> TypeModifierCollections;

        // ═══════════ 属性类型标识 ═══════════
        public AttributeType AttributeType { get; set; }

        // ═══════════ 构造函数 ═══════════
        public FloatNumeric()
        {
            TypeModifierCollections = new Dictionary<ModifierType, FloatModifierCollection>
            {
                { ModifierType.Add, add },
                { ModifierType.PctAdd, pctAdd },
                { ModifierType.FinalAdd, finalAdd },
                { ModifierType.FinalPctAdd, finalPctAdd },
            };

            // 订阅所有集合的变化事件 → 触发重算
            add.UpdateEvent += Update;
            pctAdd.UpdateEvent += Update;
            finalAdd.UpdateEvent += Update;
            finalPctAdd.UpdateEvent += Update;
        }

        // ═══════════ 核心计算 ═══════════
        public void Update()
        {
            var value1 = baseValue;
            var value2 = (value1 + add.TotalValue) * (100 + pctAdd.TotalValue) / 100;
            var value3 = (value2 + finalAdd.TotalValue) * (100 + finalPctAdd.TotalValue) / 100;
            Value = value3;
        }

        // ═══════════ 设置基础值 ═══════════
        public void SetBase(float value)
        {
            baseValue = value;
            Update();
        }
    }
}

2.5 四层计算公式详解

公式拆解

计算步骤
// 第一层：基础值
value1 = baseValue

// 第二层：基础 + 加法修改，然后乘以百分比
value2 = (value1 + add.TotalValue) × (100 + pctAdd.TotalValue) / 100

// 第三层：上一层结果 + 最终加法，再乘以最终百分比
value3 = (value2 + finalAdd.TotalValue) × (100 + finalPctAdd.TotalValue) / 100

// 最终值
Value = value3

具体计算示例

场景baseValueAddPctAddFinalAddFinalPctAdd结果
纯基础1000000 100
+装备 20 攻10020000 (100+20)×1 = 120
+Buff 50% 攻100205000 (100+20)×1.5 = 180
+光环 +10 最终1002050100 (180+10)×1 = 190
+被动 20% 最终10020501020 (180+10)×1.2 = 228
四层的设计意义
为什么不是单层 List？

Add + PctAdd（基础层）：代表装备/基础 Buff 的加成。百分比是基于「基础+Add」计算的。
→ 意义：一个 +50% Buff 只加成「基础攻击」，不会加成后续的最终加法

FinalAdd + FinalPctAdd（最终层）：代表不受基础百分比影响的加成。
→ 意义：「无视防御的固定穿透伤害」或「最终伤害减免 30%」

实际 MOBA 映射：
• 装备面板攻击力 → Add
• 「攻击力提升 50%」Buff → PctAdd
• 红 Buff 附加 50 真实伤害 → FinalAdd
• 「造成伤害提高 20%」（如输出方的终伤） → FinalPctAdd

可逆性证明

可逆性示例
// 场景：角色有 base=100，装备+20(Add)，Buff+50%(PctAdd)
// 当前值 = (100+20) * 1.5 = 180

// Buff 结束，移除 PctAdd Modifier：
pctAdd.RemoveModifier(buffModifier);  // pctAdd.TotalValue: 50 → 0
// 自动触发 Update():
// value2 = (100+20) * (100+0)/100 = 120 ← 精确回退到 Buff 前

// 对比直接减法：
// 180 - (100+20)*0.5 = 180 - 60 = 120 ← 数学上正确但需要记住原始计算
// 180 - 50% = 90 ← 错误！这是减了 180 的 50%
// 用 Modifier 集合就不会犯这种错误 — Remove 后自动重算

2.6 AttributeComponent — 属性面板

AttributeComponent 是 CombatEntity 的组件，持有该角色的所有属性值（都是 FloatNumeric 实例）。

NumericSystem — 属性操作 API

NumericSystem 关键方法
public static class NumericSystem
{
    /// 设置基础值
    public static void SetBase(FloatNumeric numeric, float value)
    {
        numeric.baseValue = value;
        numeric.Update();
    }

    /// 添加修改器
    public static void AddModifier(FloatNumeric numeric, ModifierType type, FloatModifier modifier)
    {
        numeric.TypeModifierCollections[type].AddModifier(modifier);
        // ↑ 自动触发 Collection.UpdateEvent → FloatNumeric.Update() → 重算 Value
    }

    /// 移除修改器
    public static void RemoveModifier(FloatNumeric numeric, ModifierType type, FloatModifier modifier)
    {
        numeric.TypeModifierCollections[type].RemoveModifier(modifier);
    }
}

2.7 实际使用场景分析

场景 1: Buff 修改属性

EffectAttributeSystem.cs — Enable/Disable
// Buff 生效时（Enable）— 添加 Modifier
public void Enable(AbilityEffect entity, EffectAttributeModifyComponent component)
{
    var parentEntity = entity.OwnerAbility.Parent;  // 获取挂载 Buff 的角色
    var effect = component.AttributeModifyEffect;

    // 解析配置中的数值（如 "50" 或 "30%"）
    var numericValue = component.ModifyValueFormula.Replace("%", "");
    var value = (float)ExpressionHelper.Evaluate(numericValue);

    // 创建修改器实例
    var modifier = new FloatModifier() { Value = value };

    // 找到目标属性（如 Attack）
    var numeric = AttributeSystem.GetNumeric(parentEntity as CombatEntity, effect.AttributeType);

    // 根据修改类型添加到对应层
    if (effect.ModifyType == ModifyType.Add)
        NumericSystem.AddModifier(numeric, ModifierType.FinalAdd, modifier);
    if (effect.ModifyType == ModifyType.PercentAdd)
        NumericSystem.AddModifier(numeric, ModifierType.FinalPctAdd, modifier);

    // ★ 保存 modifier 引用，用于 Disable 时移除
    component.AttributeModifier = modifier;
}

// Buff 失效时（Disable）— 移除同一个 Modifier 实例
public void Disable(AbilityEffect entity, EffectAttributeModifyComponent component)
{
    var numeric = AttributeSystem.GetNumeric(...);
    if (component.AttributeModifyEffect.ModifyType == ModifyType.Add)
        NumericSystem.RemoveModifier(numeric, ModifierType.FinalAdd, component.AttributeModifier);
    if (component.AttributeModifyEffect.ModifyType == ModifyType.PercentAdd)
        NumericSystem.RemoveModifier(numeric, ModifierType.FinalPctAdd, component.AttributeModifier);
    // ↑ 移除的是同一个实例！自动触发重算，数值精确恢复
}
完美闭环
注意 component.AttributeModifier = modifier 这行：组件保存了 Modifier 引用。
当 Buff 结束（Ability.Enable = false），EcsNode 自动调用所有 Component 的 IDisable。
EffectAttributeSystem.Disable 用保存的引用 Remove 掉 Modifier。
FloatNumeric 自动 Update()。零手动管理，零遗漏。

场景 3: 表达式解析

EffectDamageSystem.cs — 伤害公式解析
// 配置中可以写表达式，引用属性值作为参数
// 例如 DamageValueFormula = "攻击力数值 * 1.5 + 100"

private static int ParseDamage(AbilityEffect entity)
{
    var component = entity.GetComponent<EffectDamageComponent>();
    var expression = ExpressionHelper.ExpressionParser
        .EvaluateExpression(component.DamageValueFormula);

    // 注入运行时参数值
    if (expression.Parameters.ContainsKey("攻击力数值"))
    {
        expression.Parameters["攻击力数值"].Value =
            entity.OwnerEntity.GetComponent<AttributeComponent>().Attack.Value;
    }

    return (int)Math.Ceiling((float)expression.Value);
}
数据驱动的威力
配置中写 "攻击力数值 * 1.5 + 100"，运行时通过表达式引擎解析，注入实时属性值。
策划改伤害公式只需要改 Excel 配置字符串，不需要程序改代码。
这就是为什么 FloatNumeric.Value 必须随时是最新的 — 表达式随时会读取它。

2.8 设计总结

核心不变量
FloatNumeric.Value 永远等于公式计算结果。任何 Modifier 增删都立即触发重算。不存在「脏标记延迟计算」— 简单粗暴但正确。
内存开销
每个 FloatNumeric = 1 float + 4 个 List + 1 个 Dictionary。一个角色 8 个属性 ≈ 8 个 FloatNumeric 实例。对比 MOBA 中几十个角色，总计百级别实例，内存可忽略。
潜在优化点
如果大量 Modifier 同时增删（如群体 Buff），每个 Add/Remove 都触发一次 Update。可考虑批量操作后统一 Update 一次。当前实现足够用。

答案提示：第 2 题

v2 = (200 + 30) × (100 + 40) / 100 = 230 × 1.4 = 322
v3 = (322 + 15) × (100 + 10) / 100 = 337 × 1.1 = 370.7
