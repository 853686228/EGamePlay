Chapter 03

CombatEntity — 战斗实体

战斗中每个参与者（英雄、怪物、召唤物）的根实体。它本身不包含逻辑，只是组件的挂载点和子实体（Ability）的容器。理解 CombatEntity 的组成是理解整个技能释放链路的前提。

3.1 概述

核心定位
CombatEntity = 数据容器 + 能力入口。
• 作为容器：挂载 AttributeComponent（属性）、AbilityComponent（技能列表）、BuffComponent（Buff 列表）等
• 作为入口：持有 SpellAbility、DamageAbility 等 Action Ability 接口，是触发任何战斗行为的唯一入口
• 作为父节点：所有 Ability（技能/Buff）都是 CombatEntity 的子 Entity

3.2 完整源码

CombatEntity.cs
using System;
using System.Collections.Generic;
using UnityEngine;
using ECS;
using GameUtils;

namespace EGamePlay.Combat
{
    public interface IPosition
    {
        Vector3 Position { get; set; }
        float Rotation { get; set; }
    }

    /// 战斗实体 — 所有战斗参与者的根实体
    public partial class CombatEntity : EcsEntity, IPosition
    {
        // ═══════════ 基础属性 ═══════════
        public object Actor { get; set; }           // 关联的表现层对象
        public Vector3 Position { get; set; }       // 世界坐标
        public float Rotation { get; set; }         // 朝向角度
        public bool IsHero { get; set; }            // 英雄 vs 怪物标识

        // ═══════════ 行动禁制状态 ═══════════
        public ActionControlType ActionControlType { get; set; }
        // 标记当前受到的控制效果（眩晕/沉默/定身等）

        // ═══════════ Action Ability 接口 ═══════════
        // 每种战斗行为对应一个 Ability 工厂
        public IActionAbility SpellAbility { get; set; }         // 施法
        public IActionAbility DamageAbility { get; set; }        // 造成伤害
        public IActionAbility CureAbility { get; set; }          // 治疗
        public IActionAbility AddStatusAbility { get; set; }     // 施加状态
        public IActionAbility AttackAbility { get; set; }        // 普通攻击
        public IActionAbility CollisionAbility { get; set; }     // 碰撞处理
        public IActionAbility EffectAssignAbility { get; set; }  // 效果分发

        // ═══════════ 当前执行体 ═══════════
        public AbilityExecution SpellingExecution { get; set; }
        // 当前正在执行的技能（用于判断是否正在施法、可否被打断）
    }
}

3.3 挂载组件详解

Component职责关键数据创建时机
AttributeComponent 属性面板 HP、ATK、DEF、暴击率等 FloatNumeric 实例 CombatEntity 创建时
AbilityComponent 技能库 Dictionary<long, Ability> IdAbilities CombatEntity 创建时
BuffComponent Buff 库 按类型分组的活跃 Buff 列表 CombatEntity 创建时
SkillComponent 主动技能 可施放技能的子集合 CombatEntity 创建时
SpellComponent 施法管理 施法目标、方向、当前施法状态 CombatEntity 创建时
BehaviourPointComponent 行为点系统 行动点观察者列表（Trigger 订阅点） CombatEntity 创建时
ActionAbilityComponent Action 工厂注册 持有各 ActionAbility 的具体实现 CombatEntity 创建时

3.4 Action Ability 接口

CombatEntity 通过 Action Ability 接口发起战斗行为。接口只有一个方法：TryMakeAction — 尝试创建一个 Action 实体。

IActionAbility.cs
namespace EGamePlay.Combat
{
    /// Action 能力接口 — 工厂模式
    public interface IActionAbility
    {
        /// 尝试创建一个 Action 实体
        /// 返回 true 表示成功创建，out 参数为新的 Action
        bool TryMakeAction(out IActionExecute action);
    }
}

Action Ability 使用模式

使用示例
// 在 EffectDamageSystem 中造成伤害：
if (entity.OwnerEntity.DamageAbility.TryMakeAction(out var damageAction))
{
    damageAction.Target = effectAssign.Target;
    damageAction.DamageSource = DamageSource.Skill;
    DamageActionSystem.Execute(damageAction);
}

// 在 EffectAddBuffSystem 中施加 Buff：
if (entity.OwnerEntity.AddStatusAbility.TryMakeAction(out var action))
{
    action.Target = effectAssign.Target;
    action.SourceAbility = effectAssign.SourceAbility;
    AddBuffActionSystem.Execute(action);
}

// 在 AbilityTriggerSystem 中分发效果：
if (entity.OwnerEntity.EffectAssignAbility.TryMakeAction(out var effectAssign))
{
    effectAssign.AbilityEffect = abilityEffect;
    effectAssign.AssignTarget = target;
    AssignActionSystem.Execute(effectAssign);
}
为什么用工厂接口而不是直接 new
解耦：Effect System 不需要知道 DamageAction 是怎么创建的（可能需要对象池、ID 分配等）。
可拦截：TryMakeAction 可以返回 false（如沉默状态下不能施法）。
可替换：不同的 CombatEntity 可以有不同的 DamageAbility 实现（如 Boss 有特殊伤害逻辑）。

3.5 实体层级关系

CombatEntity 作为树根，所有战斗相关实体都是它的子孙节点：

Entity 树结构
CombatEntity                     // 根节点 — 一个英雄/怪物
├── Components:
│   ├── AttributeComponent       // HP, ATK, DEF...
│   ├── AbilityComponent         // 技能索引表
│   ├── BuffComponent            // Buff 索引表
│   ├── SkillComponent           // 主动技能子集
│   ├── SpellComponent           // 施法状态
│   └── BehaviourPointComponent  // 行为点观察者
│
├── Children (Abilities):
│   ├── Ability [技能1: 火球术]
│   │   ├── AbilityEffect [效果: 200 伤害]
│   │   │   └── EffectDamageComponent
│   │   ├── AbilityEffect [效果: 施加燃烧]
│   │   │   └── EffectAddBuffComponent
│   │   └── AbilityTrigger [触发: 碰撞时]
│   │       └── BehaviourObserveComponent
│   │
│   ├── Ability [技能2: 治疗术]
│   │   └── ...
│   │
│   ├── Ability [Buff: 燃烧 DOT]  // IsBuff = true
│   │   ├── AbilityEffect [效果: 每秒 50 伤害]
│   │   │   └── EffectDamageComponent
│   │   ├── AbilityTrigger [触发: 每 1 秒]
│   │   │   └── TimeStateObserveComponent
│   │   └── AbilityDurationComponent  // 5 秒后自动移除
│   │
│   └── Ability [被动: 反伤]
│       ├── AbilityEffect [效果: 反弹 10% 伤害]
│       └── AbilityTrigger [触发: 受伤时]
│           └── BehaviourObserveComponent
│
└── SpellingExecution:
    └── AbilityExecution [当前正在释放的技能]  // 临时子实体
层级 = 所有权 = 生命周期
当 CombatEntity 被销毁（角色死亡）时，EcsObject.Destroy(entity) 会递归销毁整棵子树：
所有 Ability → 所有 Effect/Trigger → 所有 Component → 自动清理

Buff 同理：当 Buff 到期（AbilityDurationSystem 触发），调用 BuffSystem.Remove() 删除该 Ability 子节点，其所有 Effect/Trigger 自动跟着清理。零手动管理。

3.6 创建与初始化

CombatEntity 创建流程（伪代码）
// 在 CombatEntitySystem.Awake 中：
public void Awake(CombatEntity entity)
{
    // 1. 添加核心组件
    entity.AddComponent<AttributeComponent>();
    entity.AddComponent<AbilityComponent>();
    entity.AddComponent<BuffComponent>();
    entity.AddComponent<SkillComponent>();
    entity.AddComponent<SpellComponent>();
    entity.AddComponent<BehaviourPointComponent>();

    // 2. 初始化 Action Ability 接口
    entity.SpellAbility = new SpellActionAbility(entity);
    entity.DamageAbility = new DamageActionAbility(entity);
    entity.CureAbility = new CureActionAbility(entity);
    entity.AddStatusAbility = new AddBuffActionAbility(entity);
    entity.AttackAbility = new AttackActionAbility(entity);
    entity.CollisionAbility = new CollisionActionAbility(entity);
    entity.EffectAssignAbility = new EffectAssignActionAbility(entity);

    // 3. 初始化属性基础值（从配置加载）
    var attr = entity.GetComponent<AttributeComponent>();
    NumericSystem.SetBase(attr.HealthPointMax, 1000);
    NumericSystem.SetBase(attr.HealthPoint, 1000);
    NumericSystem.SetBase(attr.Attack, 100);
    NumericSystem.SetBase(attr.Defense, 50);
    // ...

    // 4. 挂载初始技能
    foreach (var skillConfig in heroConfig.Skills)
    {
        AbilitySystem.Attach(entity, skillConfig);
    }
}

3.7 设计总结

CombatEntity 不做逻辑
它只有属性声明，没有方法。所有行为都通过 Action Ability 接口代理到对应的 System。这保证了数据与逻辑的分离。
partial class 扩展
CombatEntity 声明为 partial class，允许不同模块在不修改原文件的情况下扩展属性。这在大型项目多人协作中很有用。
Position 接口
实现 IPosition 接口让碰撞/移动系统能以统一方式访问位置信息。Position/Rotation 是逻辑层坐标，由 ViewSystem 同步到 Unity Transform。
验收标准
1. CombatEntity 的 Children 里存的是什么？
2. DamageAbility.TryMakeAction() 做了什么？谁调用它？
3. 一个角色死亡时，它身上的 Buff 如何被清理？
4. AbilityComponent 和 BuffComponent 存的都是 Ability，为什么要分两个 Component？
5. ActionControlType 在哪里被设置？被谁读取？
