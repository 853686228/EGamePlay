Chapter 09

Configuration Pipeline — 配置管线

策划如何配置一个技能？从 Excel 表格到运行时数据结构，从 ScriptableObject 到时间轴编辑器。本章解析 EGamePlay 的双轨配置架构：Excel 管参数，ScriptableObject 管结构。

9.1 概述

核心问题
一个 MOBA 技能的配置信息包含：
• 参数型数据：CD、伤害值、范围、持续时间 → 策划频繁调整
• 结构型数据：几个效果？什么触发条件？时间轴如何编排？ → 设计阶段确定

用单一方案管理两类数据会出问题：
• 纯 Excel → 无法表达嵌套结构（效果列表、触发条件树）
• 纯 ScriptableObject → 策划改个 CD 要打开 Unity、找资源、改值、保存

解决方案：双轨。

9.2 双轨架构

Excel — 参数轨
管什么：数值参数
工具：Excel → 导出工具 → JSON → C# 类
谁用：策划日常调参
频率：每天多次
产物：AbilityConfig 类（运行时配置）
ScriptableObject — 结构轨
管什么：效果组合、触发逻辑、时间轴
工具：Unity Inspector + 自定义编辑器
谁用：技能设计师
频率：设计阶段
产物：AbilityConfigObject + ExecutionObject

两轨如何关联

AbilityConfig (Excel)
Id = 1001, Name = "火球术", CD = 8s
↓ Id 关联
AbilityConfigObject (SO)
Id = 1001, Effects = [DamageEffect, AddStatusEffect]
↓ ExecutionObject 引用
ExecutionObject (SO)
Timeline: 0.3s 播特效, 0.5s 发弹道, 0.8s 碰撞检测
Id 是两轨的桥梁
AbilityConfig.Id 和 AbilityConfigObject.Id 相同时，运行时自动关联。
策划在 Excel 改 CD = 6s → 导出 → 运行时 AbilityConfig 更新 → Ability 读到新 CD。
不需要碰 ScriptableObject，不需要打开 Unity。

9.3 AbilityConfig — 运行时配置（Excel 产物）

由 Excel 导出工具自动生成的 C# 类。每一行 Excel 数据对应一个 AbilityConfig 实例。

AbilityConfig.cs — 自动生成的配置类
namespace EGamePlay.Combat
{
    /// 技能/Buff 的基础参数配置（来自 Excel）
    public class AbilityConfig
    {
        // ═══════════ 标识 ═══════════
        public int Id;              // 唯一 ID（与 ConfigObject.Id 关联）
        public string KeyName;     // 代码引用键名（如 "Skill_Fireball"）
        public string Name;        // 显示名称（如 "火球术"）

        // ═══════════ 类型 ═══════════
        public string Type;        // "Skill" 或 "Buff"

        // ═══════════ 目标规则 ═══════════
        public string TargetGroup;  // "己方" / "敌方" / "自身"
        public string TargetSelect; // "碰撞检测" / "条件指定" / "手动指定"

        // ═══════════ 技能参数 ═══════════
        public float Cooldown;     // 冷却时间（秒）
        public float Duration;     // 持续时间（Buff 用，技能不填）

        // ═══════════ Buff 专用 ═══════════
        public string BuffType;    // Buff 类型分类（增益/减益/控制）
        public int StatusSlot;    // 状态槽位（同槽位互斥/刷新）
        public bool CanStack;     // 是否可叠加
    }
}

字段详解

字段类型说明Excel 示例
Idint全局唯一标识，关联 ConfigObject1001
KeyNamestring代码中引用的键Skill_Fireball
NamestringUI 显示名火球术
Typestring区分技能和 BuffSkill / Buff
TargetGroupstring目标阵营敌方
TargetSelectstring目标获取方式碰撞检测
CooldownfloatCD 秒数8
DurationfloatBuff 持续秒数5
BuffTypestring增益/减益/控制减益
StatusSlotint互斥槽1
CanStackbool是否可叠层TRUE
TargetSelect 的三种模式
碰撞检测：由 AbilityItem（弹道/AOE 区域）的物理碰撞决定命中谁。如火球飞行中碰到的第一个敌人。
条件指定：运行时按条件自动选取。如「范围内 HP 最低的友方」。
手动指定：玩家主动选择目标。如指向性技能点击敌方英雄。

9.4 AbilityConfigObject — ScriptableObject 结构配置

AbilityConfigObject 是 Unity ScriptableObject，在 Inspector 中可视化编辑。它定义技能/Buff 的效果组合和触发逻辑。

AbilityConfigObject.cs — 核心结构
namespace EGamePlay.Combat
{
    /// 技能的目标影响类型
    public enum AffectTargetType
    {
        SelfTeam,       // 己方
        EnemyTeam,      // 敌方
        Self,           // 仅自身
    }

    /// 目标选取类型
    public enum TargetSelectType
    {
        CollisionSelect,    // 碰撞检测
        ConditionSelect,    // 条件指定
        ManualSelect,       // 手动指定
    }

    /// 技能/Buff 的结构配置（ScriptableObject）
    [CreateAssetMenu(fileName = "AbilityConfig", menuName = "能力/能力配置")]
    public class AbilityConfigObject : ScriptableObject
    {
        // ═══════════ 关联 ID ═══════════
        public int Id;  // 与 AbilityConfig.Id 一一对应

        // ═══════════ 目标规则 ═══════════
        public AffectTargetType AffectTargetType;
        public TargetSelectType TargetSelectType;

        // ═══════════ 效果列表 ═══════════
        // 一个技能可以有多个效果（如：造成伤害 + 施加减速 + 回血）
        [SerializeReference]
        public List<Effect> Effects = new();

        // ═══════════ 触发配置 ═══════════
        // 定义每个效果在什么条件下触发
        public List<TriggerConfig> TriggerActions = new();

        // ═══════════ 执行体引用 ═══════════
        // 指向 ExecutionObject（时间轴配置）
        public ExecutionObject ExecutionObject;

        // ═══════════ 参数字典 ═══════════
        // 用于 Buff 参数传递（源技能 → Buff）
        public Dictionary<string, string> Parameters;
    }
}
[SerializeReference] 的魔力
Effects 列表用 [SerializeReference] 而不是 [SerializeField]：
• 允许列表中存放 Effect 的不同子类（DamageEffect、CureEffect、...）
• Unity Inspector 会显示每个效果的具体类型字段
• 这实现了「一个技能多个不同类型的效果」的组合配置

9.5 Effect 配置类

Effect 是 AbilityConfigObject 中效果列表的元素。基类 Effect 有 6 个子类，每个代表一种效果类型。

继承体系

Effect 类层级
Effect (abstract base)
├── DamageEffect            // 造成伤害
├── CureEffect              // 治疗恢复
├── AddStatusEffect         // 施加 Buff/Debuff
├── AttributeModifyEffect   // 属性修改
├── ActionControlEffect     // 行动控制（眩晕/沉默）
└── ShieldDefenseEffect     // 护盾

DamageEffect — 伤害效果配置

DamageEffect.cs
namespace EGamePlay.Combat
{
    [Serializable]
    public class DamageEffect : Effect
    {
        /// 伤害值公式（字符串表达式）
        /// 支持参数引用，如 "攻击力数值 * 1.5 + 100"
        /// 运行时由 ExpressionHelper 解析，注入属性值
        public string DamageValueFormula;

        /// 伤害修饰器列表（可选）
        /// 如：忽视防御百分比、附加元素伤害等
        public List<DamageDecorator> Decorators = new();
    }
}

AttributeModifyEffect — 属性修改效果配置

AttributeModifyEffect.cs
namespace EGamePlay.Combat
{
    /// 修改方式
    public enum ModifyType
    {
        BaseValue,    // 直接修改基础值（永久）
        Add,          // 加法修改器（可逆）
        PercentAdd,   // 百分比修改器（可逆）
    }

    [Serializable]
    public class AttributeModifyEffect : Effect
    {
        /// 修改哪个属性
        public AttributeType AttributeType;  // Attack/Defense/MoveSpeed/...

        /// 修改方式

        /// 修改数值（或百分比值）
        /// 如 "50" = +50攻击力, 或 "30" = +30%(PercentAdd)
        public string NumericValue;
    }
}

ShieldDefenseEffect.cs
namespace EGamePlay.Combat
{
    [Serializable]
    public class ShieldDefenseEffect : Effect
    {
        /// 护盾值（吸收伤害量）
        /// 支持表达式，如 "最大生命值 * 0.15"
        public string ShieldValue;

}
Effect 配置 vs Effect 实体 — 不要混淆
配置层（本章）：DamageEffect 是配置数据类，存在 ScriptableObject 中，策划编辑。
运行时层（Ch05）：AbilityEffect 是 EcsEntity，持有 EffectDamageComponent，被 System 驱动。

关系：AbilityEffect.EffectConfig = DamageEffect（运行时实体引用配置数据）

9.6 TriggerConfig — 触发配置

TriggerConfig 定义效果在什么条件下执行。每个 TriggerConfig 关联一组 Effect，描述「什么时候」执行「哪些效果」。

TriggerConfig.cs — 完整结构
namespace EGamePlay.Combat
{
    /// 触发方式
    public enum TriggerType
    {
        AutoTrigger,    // 自动触发（由系统驱动）
        ManualTrigger,  // 手动触发（由时间轴 Clip 驱动）
    }

    /// 自动触发的子类型
    public enum AutoTriggerType
    {
        Instant,    // 立即（Ability 激活时）
        Action,     // 行动点（某个 ActionPoint 触发时）
        Condition,  // 条件（时间/状态/事件满足时）
    }

    [Serializable]
    public class TriggerConfig
    {
        // ═══════════ 触发方式 ═══════════
        public TriggerType TriggerType;

        // ═══════════ 自动触发配置 ═══════════
        public AutoTriggerType AutoTriggerType;

        /// Action 触发时：在哪个行动点触发
        /// 如 AfterReceiveApply = 受到伤害后触发
        public ActionPointType ActionPointType;

        /// Condition 触发时：条件类型
        /// 如 WhenHPBelowPercent / WhenTimeInterval
        public TimeStateEventType ConditionType;

        // ═══════════ 条件参数 ═══════════
        /// 状态检查列表
        /// 如 ["HP < 30%", "HasBuff_Shield == false"]
        public List<string> StateCheckList = new();

        // ═══════════ 要触发的效果 ═══════════
        /// 当触发条件满足时，执行哪些效果
        /// 每个 TriggerEffect 包含效果索引 + 应用方式
        public List<TriggerEffect> TriggerEffects = new();
    }

    [Serializable]
    public class TriggerEffect
    {
        /// 效果在 Effects 列表中的索引
        public int EffectIndex;

        /// 效果应用方式
        public EffectApplyType EffectApplyType;
    }

    /// 效果应用方式
    public enum EffectApplyType
    {
        AllTargets,     // 对所有命中目标
        Creator,        // 对施法者自身
        FirstTarget,    // 仅对第一个目标
    }
}

触发类型决策树

TriggerTypeAutoTriggerType何时触发MOBA 实例
AutoTriggerInstantBuff 挂载时立即红 Buff 立即加攻速
AutoTriggerAction特定 ActionPoint 时荆棘甲：受到伤害后反伤
AutoTriggerCondition状态/时间条件满足时斩杀：目标 HP < 30% 时额外伤害
ManualTrigger—时间轴 Clip 手动触发火球术：0.5s 时发射弹道
TriggerConfig 是 Ch06 触发系统的配置源
运行时，AbilitySystem.Awake 读取每个 TriggerConfig，创建对应的 AbilityTrigger 实体：
• Instant → 直接执行效果
• Action → 注册到 CombatEntity 的 ActionPoint 回调
• Condition → 创建定时器/条件监听器
• ManualTrigger → 由 ExecutionObject 的 Timeline Clip 触发

9.7 ExecutionObject — 执行时间轴配置

ExecutionObject 是 ScriptableObject，定义技能释放的时间轴（参见 Ch07 详细解析）。这里总结其配置相关字段。

ExecutionObject.cs — 配置字段摘要
[CreateAssetMenu(fileName = "Execution", menuName = "能力/Execution")]
public class ExecutionObject : ScriptableObject
{
    // ═══════════ 时间轴总时长 ═══════════
    public float TotalTime;  // 执行体总时长（秒）

    // ═══════════ 时间轴片段 ═══════════
    // 每个 Clip 在特定时间点执行特定操作
    [SerializeReference]
    public List<ExecutionClip> ExecutionClips = new();

    // ═══════════ Clip 类型 ═══════════
    // ExecutionClip 子类：
    // • SpawnItemClip      — 生成弹道/AOE 区域
    // • TriggerApplyClip   — 手动触发效果应用
    // • PlayAnimationClip  — 播放动画
    // • PlayEffectClip     — 播放特效
    // • PlayAudioClip      — 播放音效
}
ExecutionObject 与 AbilityConfigObject 的关系
AbilityConfigObject.ExecutionObject 引用了对应的 ExecutionObject。
• AbilityConfigObject 定义「这个技能有什么效果」
• ExecutionObject 定义「这些效果在什么时间以什么方式表现出来」

同一个 AbilityConfigObject 可以搭配不同的 ExecutionObject（如普通施法 vs 蓄力施法的不同时间轴）。

9.8 Excel → Runtime 管线

策划修改 Excel 后，数据如何流入运行时？

完整管线步骤

Step 1: 编辑 Excel
策划在 AbilityConfig.xlsx 中增/改行
↓
Step 2: 导出配置
Unity 菜单 → Tools/导出配置
↓
Step 3: 生成 JSON

↓

// 游戏启动时
public static class AppStatic
{
    public static ConfigManager Config;

    public static void Init()
    {
        Config = new ConfigManager();
        // 从 Resources/Config/ 加载所有 JSON
        // 反序列化为对应的 Config 类实例
        // 建立 Id → Config 的 Dictionary 索引
    }
}

// 使用时
var config = AppStatic.Config.Get<AbilityConfig>(1001);
// → 返回 Id=1001 的火球术配置
// config.Cooldown = 8f
// config.Name = "火球术"
为什么中间有 JSON 层？
• 版本控制友好：JSON 是文本，Git diff 可读
• 热更新支持：运行时可以从服务器下发 JSON 覆盖本地配置
• 跨平台：Excel 是 Windows 工具，JSON 所有平台都能读
• 测试方便：单元测试直接构造 JSON，不依赖 Excel

9.9 完整流程：创建一个新技能

假设我们要创建「寒冰箭」：指向性弹道，命中后造成伤害 + 施加减速 Debuff。

Step 1: 在 Excel 添加参数行

AbilityConfig.xlsx — 新增行

| 1002 | Skill_IceArrow | 寒冰箭 | Skill | 敌方        | 手动指定     | 6        |          |

9.10 设计总结

Excel = 快速迭代
策划改数值不需要打开 Unity。Excel 编辑 → 一键导出 → 游戏内生效。数值平衡迭代效率极高。所有数值参数（CD、伤害系数、持续时间）都走这条路。
ScriptableObject = 可视化设计
效果组合、触发逻辑、时间轴编排 — 这些结构型数据在 Inspector 中可视化配置。技能设计师拖拖拽拽就能组合出复杂技能，不需要写代码。
Id 是唯一桥梁
两轨通过 Id 字段关联。Excel 中的 AbilityConfig.Id 必须与 ScriptableObject 中的 AbilityConfigObject.Id 相同。这是唯一的耦合点 — 简单但有效。
已知局限
• Id 关联靠人工保证一致性（无自动校验）
• Excel 导出是全量导出（改一行重新生成全部）
• ScriptableObject 无法热更新（需要 AssetBundle 方案）
• 无配置校验工具（填错公式只有运行时才报错）

2. AbilityConfig（Excel 产物）和 AbilityConfigObject（SO）的职责分别是什么？
