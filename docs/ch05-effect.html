Chapter 05

AbilityEffect — 效果系统

Effect 是 Ability 的「做什么」。一个技能/Buff 的实际效果——伤害、治疗、加 Buff、修改属性、控制行为、护盾——全部通过 AbilityEffect 子实体 + 对应 Component + System 来实现。

5.1 概述

核心架构
AbilityEffect 是一个 Entity，挂载在 Ability 下。它通过 组件组合 声明自己的类型：
• 挂了 EffectDamageComponent → 这是一个伤害效果
• 挂了 EffectCureComponent → 这是一个治疗效果
• 挂了 EffectAddBuffComponent → 这是一个施加 Buff 效果

每种 Component 有对应的 System 处理具体逻辑。Effect 本身不包含任何业务代码。

关键模式：Effect 是配置声明，Action 是运行时执行。 Effect 描述「能造成什么」，Action 是「真正造成了什么」。

5.2 AbilityEffect 实体

AbilityEffect.cs — 完整源码
namespace EGamePlay.Combat
{
    public struct TriggerContext
    {
        public AbilityTrigger AbilityTrigger;   // 触发此效果的 Trigger
        public EcsEntity TriggerSource;        // 触发源
        public AbilityItem AbilityItem;        // 技能投射物（如子弹）
        public EcsEntity Target;              // 效果目标
    }

    /// 能力效果，如伤害、治疗、施加状态等
    public partial class AbilityEffect : EcsEntity
    {
        public string Name { get; set; }
        public Ability OwnerAbility => GetParent<Ability>();
        public CombatEntity OwnerEntity => OwnerAbility.OwnerEntity;
        public EcsEntity ParentEntity => OwnerAbility.ParentEntity;
        public Effect EffectConfig { get; set; }  // 效果配置基类引用
    }
}
设计洞察 — 向上导航
OwnerAbility 通过 GetParent<Ability>() 获得（层级关系）
OwnerEntity 通过 OwnerAbility.OwnerEntity 再向上跳一层

这意味着从任何一个 AbilityEffect 实例，你可以沿着父子链向上找到持有它的 Ability 和 CombatEntity。无需额外存储引用。

TriggerContext 是一个值类型 struct，用于在触发时传递上下文信息——谁触发的、对谁生效、通过什么载体（子弹/范围）。

5.3 AbilityEffectSystem.Create — 工厂方法

Create 是 AbilityEffect 的工厂方法。它根据配置类型动态挂载对应的 Component，实现了配置驱动的组件组合。

AbilityEffectSystem.cs — Create 工厂
public static AbilityEffect Create(Ability ability, Effect effectConfig)
{
    var abilityEffect = ability.AddChild<AbilityEffect>();
    abilityEffect.EffectConfig = effectConfig;
    abilityEffect.Name = effectConfig.GetType().Name;

    /// 行动控制
    if (effectConfig is ActionControlEffect)
        abilityEffect.AddComponent<EffectActionControlComponent>();

    /// 属性修改
    if (effectConfig is AttributeModifyEffect)
        abilityEffect.AddComponent<EffectAttributeModifyComponent>();

    /// 伤害效果
    if (effectConfig is DamageEffect)
        abilityEffect.AddComponent<EffectDamageComponent>();

    /// 治疗效果
    if (effectConfig is CureEffect)
        abilityEffect.AddComponent<EffectCureComponent>();

    /// 护盾防御效果
    if (effectConfig is ShieldDefenseEffect)
        abilityEffect.AddComponent<EffectShieldDefenseComponent>();

    /// 施加状态效果
    if (effectConfig is AddStatusEffect)
        abilityEffect.AddComponent<EffectAddBuffComponent>();

    /// 效果装饰器
    var decorators = abilityEffect.EffectConfig.Decorators;
    if (decorators != null && decorators.Count > 0)
        abilityEffect.AddComponent<EffectDecoratorComponent>();

    return abilityEffect;
}
类型判断用 is 而非字符串
注意这里用的是 C# 的 is 类型检测（effectConfig is DamageEffect），而不是字符串比较。
配置类型是继承自 Effect 基类的具体子类，编译期类型安全。

这与 AbilitySystem 中用字符串 "Buff"/"Skill" 判断不同 — 因为 Effect 类型是由程序员定义的代码类型，而 Skill/Buff 的分类是策划配表决定的。
用 if 不用 switch/else if 的原因
注意这里不是 else if 而是独立的 if。这意味着一个 Effect 理论上可以同时挂多个 Component（如果配置类型有多重继承关系）。虽然当前配置类型不会同时满足多个条件，但这种写法保留了扩展性。

5.4 TriggerApply — 效果分发

当 Trigger 触发时，调用 AbilityEffectSystem.TriggerApply。它遍历 AbilityEffect 的所有 Component，找到对应的 System 调用其 TriggerApply。

AbilityEffectSystem.cs — TriggerApply 分发
public static void TriggerApply(EffectAssignAction effectAssign)
{
    var abilityEffect = effectAssign.AbilityEffect;

    // 遍历 AbilityEffect 身上的所有组件
    foreach (var item in effectAssign.AbilityEffect.Components.Values)
    {
        if (item is EffectActionControlComponent)
            EffectActionControlSystem.TriggerApply(abilityEffect, effectAssign);

        if (item is EffectAddBuffComponent)
            EffectAddBuffSystem.TriggerApply(abilityEffect, effectAssign);

        if (item is EffectAttributeModifyComponent)
            EffectAttributeSystem.TriggerApply(abilityEffect, effectAssign);

        if (item is EffectCureComponent)
            EffectCureSystem.TriggerApply(abilityEffect, effectAssign);

        if (item is EffectDamageComponent)
            EffectDamageSystem.TriggerApply(abilityEffect, effectAssign);

        if (item is EffectShieldDefenseComponent)
            EffectShieldSystem.TriggerApply(abilityEffect, effectAssign);
    }
}
EffectAssignAction 是什么
EffectAssignAction 是一个 Action 实体（Ch08 详述），携带了触发上下文：
• .AbilityEffect — 哪个 Effect 被触发
• .Target — 效果作用目标
• .SourceAbility — 来源 Ability

它是 Effect（配置声明）和 Action（实际执行）之间的桥梁。

5.5 EffectDamageSystem — 伤害效果

伤害是最常用的效果类型。它解析公式计算伤害值，然后通过 DamageAbility 创建 DamageAction 执行实际伤害。

EffectDamageSystem.cs — 完整源码
public class EffectDamageSystem : AComponentSystem<AbilityEffect, EffectDamageComponent>,
    IAwake<AbilityEffect, EffectDamageComponent>
{
    public void Awake(AbilityEffect entity, EffectDamageComponent component)
    {
        component.DamageEffect = entity.EffectConfig as DamageEffect;
        component.DamageValueFormula = component.DamageEffect.DamageValueFormula;
    }

    public static int GetDamageValue(AbilityEffect entity)
    {
        return ParseDamage(entity);
    }

    private static int ParseDamage(AbilityEffect entity)
    {
        var component = entity.GetComponent<EffectDamageComponent>();
        var expression = ExpressionHelper.ExpressionParser
            .EvaluateExpression(component.DamageValueFormula);

        // 注入运行时属性值到表达式参数
        if (expression.Parameters.ContainsKey("攻击力数值"))
        {
            expression.Parameters["攻击力数值"].Value =
                entity.OwnerEntity.GetComponent<AttributeComponent>().Attack.Value;
        }

        return (int)System.Math.Ceiling((float)expression.Value);
    }

    public static void TriggerApply(AbilityEffect entity, EffectAssignAction effectAssign)
    {
        var component = entity.GetComponent<EffectDamageComponent>();
        if (entity.OwnerEntity.DamageAbility.TryMakeAction(out var damageAction))
        {
            damageAction.SourceAssignAction = effectAssign;
            damageAction.Target = effectAssign.Target;
            damageAction.DamageSource = DamageSource.Skill;
            DamageActionSystem.Execute(damageAction);
        }
    }
}
公式驱动的伤害计算
DamageValueFormula 是配置中的字符串表达式（如 "攻击力数值 * 2.5 + 100"）。
ParseDamage 在运行时解析表达式、注入当前属性值、计算结果。

这意味着策划可以在 Excel 中写任意伤害公式，无需程序修改代码。
属性值来自 FloatNumeric.Value（Ch02），已经包含所有 Buff 加成。

5.6 EffectCureSystem — 治疗效果

EffectCureSystem.cs — 完整源码
public class EffectCureSystem : AComponentSystem<AbilityEffect, EffectCureComponent>,
    IAwake<AbilityEffect, EffectCureComponent>
{
    public void Awake(AbilityEffect entity, EffectCureComponent component)
    {
        component.CureEffect = entity.EffectConfig as CureEffect;
        component.CureValueProperty = component.CureEffect.CureValueFormula;
    }

    public static int GetCureValue(AbilityEffect entity)
    {
        return ParseValue(entity);
    }

    private static int ParseValue(AbilityEffect entity)
    {
        var expression = ExpressionHelper.ExpressionParser
            .EvaluateExpression(entity.GetComponent<EffectCureComponent>().CureValueProperty);

        if (expression.Parameters.ContainsKey("生命值数值"))
        {
            expression.Parameters["生命值数值"].Value =
                entity.OwnerEntity.GetComponent<AttributeComponent>().HealthPoint.Value;
        }

        return (int)System.Math.Ceiling((float)expression.Value);
    }

    public static void TriggerApply(AbilityEffect entity, EffectAssignAction effectAssign)
    {
        if (entity.OwnerEntity.CureAbility.TryMakeAction(out var action))
        {
            action.SourceAssignAction = effectAssign;
            action.Target = effectAssign.Target;
            CureActionSystem.Execute(action);
        }
    }
}
治疗与伤害的对称结构
EffectCureSystem 和 EffectDamageSystem 结构完全对称：
• Awake：缓存配置引用 + 公式字符串
• ParseValue：解析表达式 + 注入属性值
• TriggerApply：创建 Action + Execute

区别仅在于使用的 Ability（CureAbility vs DamageAbility）和 ActionSystem（CureActionSystem vs DamageActionSystem）。

5.7 EffectAddBuffSystem — 施加状态

EffectAddBuffSystem.cs — 完整源码
public class EffectAddBuffSystem : AComponentSystem<AbilityEffect, EffectAddBuffComponent>,
    IAwake<AbilityEffect, EffectAddBuffComponent>
{
    public void Awake(AbilityEffect entity, EffectAddBuffComponent component)
    {
        component.AddStatusEffect = entity.EffectConfig as AddStatusEffect;
        component.Duration = component.AddStatusEffect.Duration;
    }

    public static int GetNumericValue(AbilityEffect entity)
    {
        return 1;
    }

    public static void TriggerApply(AbilityEffect entity, EffectAssignAction effectAssign)
    {
        if (entity.OwnerEntity.AddStatusAbility.TryMakeAction(out var action))
        {
            action.SourceAssignAction = effectAssign;
            action.Target = effectAssign.Target;
            action.SourceAbility = effectAssign.SourceAbility;
            AddBuffActionSystem.Execute(action);
        }
    }

    public static IActionExecute GetActionExecution(AbilityEffect entity)
    {
        if (entity.OwnerEntity.AddStatusAbility.TryMakeAction(out var action))
        {
            return action;
        }
        return null;
    }
}
Buff 施加的递归性
EffectAddBuff 是「能力施加能力」的实现。一个技能（Ability）的 Effect 可以给目标添加一个 Buff（另一个 Ability）。
这里的 AddStatusAbility 是 CombatEntity 上的工厂 Ability，专门负责创建「施加 Buff」的 Action。

整条链路： 技能 Ability → Effect → AddBuffAction → 目标身上 AbilitySystem.Attach() → 新 Buff Ability

5.8 EffectAttributeSystem — 属性修改

属性修改是 Buff 最常见的效果。它在 Enable 时添加 Modifier，Disable 时移除 Modifier，利用 FloatNumeric 的可逆性实现精确回退。

EffectAttributeSystem.cs — 完整源码
public class EffectAttributeSystem : AComponentSystem<AbilityEffect, EffectAttributeModifyComponent>,
    IAwake<AbilityEffect, EffectAttributeModifyComponent>,
    IEnable<AbilityEffect, EffectAttributeModifyComponent>,
    IDisable<AbilityEffect, EffectAttributeModifyComponent>
{
    public void Awake(AbilityEffect entity, EffectAttributeModifyComponent component)
    {
        component.AttributeModifyEffect = entity.EffectConfig as AttributeModifyEffect;
        component.ModifyValueFormula = component.AttributeModifyEffect.NumericValue;
    }

    public void Enable(AbilityEffect entity, EffectAttributeModifyComponent component)
    {
        var parentEntity = entity.OwnerAbility.Parent;
        var attributeModifyEffect = component.AttributeModifyEffect;
        var numericValue = component.ModifyValueFormula;
        numericValue = numericValue.Replace("%", "");

        var expression = ExpressionHelper.ExpressionParser.EvaluateExpression(numericValue);
        var value = (float)expression.Value;
        var numericModifier = new FloatModifier() { Value = value };

        var attributeType = attributeModifyEffect.AttributeType;
        var numeric = AttributeSystem.GetNumeric(parentEntity as CombatEntity, attributeType);

        if (attributeModifyEffect.ModifyType == ModifyType.BaseValue)
        {
            NumericSystem.SetBase(numeric, value);
        }
        if (attributeModifyEffect.ModifyType == ModifyType.Add)
        {
            NumericSystem.AddModifier(numeric, ModifierType.FinalAdd, numericModifier);
        }
        if (attributeModifyEffect.ModifyType == ModifyType.PercentAdd)
        {
            NumericSystem.AddModifier(numeric, ModifierType.FinalPctAdd, numericModifier);
        }

        component.AttributeModifier = numericModifier;  // ★ 保存引用用于 Disable 时移除
    }

    public void Disable(AbilityEffect entity, EffectAttributeModifyComponent component)
    {
        var parentEntity = entity.OwnerAbility.Parent;
        var attributeType = component.AttributeModifyEffect.AttributeType;
        var numeric = AttributeSystem.GetNumeric(parentEntity as CombatEntity, attributeType);

        if (component.AttributeModifyEffect.ModifyType == ModifyType.Add)
        {
            NumericSystem.RemoveModifier(numeric, ModifierType.FinalAdd,
                component.AttributeModifier);
        }
        if (component.AttributeModifyEffect.ModifyType == ModifyType.PercentAdd)
        {
            NumericSystem.RemoveModifier(numeric, ModifierType.FinalPctAdd,
                component.AttributeModifier);
        }
    }

    public static void TriggerApply(AbilityEffect entity, EffectAssignAction effectAssign)
    {
        entity.GetComponent<EffectAttributeModifyComponent>().Enable = true;
    }
}
完美闭环 — Enable/Disable 对称性
这是整套 ECS 架构的精髓展示：

Enable 时： 解析配置 → 创建 FloatModifier → AddModifier → 数值生效 → 保存 Modifier 引用
Disable 时： 取出保存的 Modifier 引用 → RemoveModifier → 数值精确恢复

Buff 结束 → Ability.Enable = false → 级联 → AbilityEffect.Enable = false → 级联 → Component.Enable = false → 触发 EffectAttributeSystem.Disable → 移除 Modifier。

整条链自动驱动，零手动管理。

5.9 EffectActionControlSystem — 行为控制

行为控制效果（眩晕、沉默、击飞等）在 Enable/Disable 时通知 BuffSystem 重新计算角色的控制状态。

EffectActionControlSystem.cs — 完整源码
public class EffectActionControlSystem : AComponentSystem<AbilityEffect, EffectActionControlComponent>,
    IAwake<AbilityEffect, EffectActionControlComponent>,
    IEnable<AbilityEffect, EffectActionControlComponent>,
    IDisable<AbilityEffect, EffectActionControlComponent>
{
    public void Awake(AbilityEffect entity, EffectActionControlComponent component)
    {
        component.ActionControlEffect = entity.EffectConfig as ActionControlEffect;
    }

    public void Enable(AbilityEffect entity, EffectActionControlComponent component)
    {
        BuffSystem.OnBuffChanged(entity.Parent.Parent as CombatEntity,
            entity.GetParent<Ability>());
    }

    public void Disable(AbilityEffect entity, EffectActionControlComponent component)
    {
        BuffSystem.OnBuffChanged(entity.Parent.Parent as CombatEntity,
            entity.GetParent<Ability>());
    }

    public static void TriggerApply(AbilityEffect entity, EffectAssignAction effectAssign)
    {
        entity.GetComponent<EffectActionControlComponent>().Enable = true;
    }
}
BuffSystem.OnBuffChanged 的作用
Enable 和 Disable 都调用同一个方法 BuffSystem.OnBuffChanged。这个方法会重新遍历角色身上所有 Buff 的 ActionControlEffect，计算当前的控制状态（是否被眩晕/沉默/禁锢等）。

这是一种声明式的状态管理 — 不关心「加了什么减了什么」，每次变更都从头算一遍「当前总共有什么控制」。简单粗暴但绝对正确。

5.10 EffectShieldSystem — 护盾效果

EffectShieldSystem.cs — 完整源码
public class EffectShieldSystem : AComponentSystem<AbilityEffect, EffectShieldDefenseComponent>,
    IAwake<AbilityEffect, EffectShieldDefenseComponent>
{
    public void Awake(AbilityEffect entity, EffectShieldDefenseComponent component)
    {
        component.ShieldDefenseEffect = entity.EffectConfig as ShieldDefenseEffect;
    }

    public static void TriggerApply(AbilityEffect entity, EffectAssignAction effectAssign)
    {
        effectAssign.Target.AddComponent<AbilityItemShieldComponent>();
    }
}
护盾的组件式实现
护盾效果最简单 — TriggerApply 时直接给目标 Entity 添加一个 AbilityItemShieldComponent。
DamageActionSystem 在执行伤害时会检查目标是否有这个组件，如果有则先扣护盾值。

这是 ECS 思想的典型应用：通过「有没有某个组件」来判断状态，而不是维护一个 bool 字段。

5.11 EffectDecoratorSystem — 效果装饰器

装饰器是对已有效果的增强/修改。例如「伤害随命中目标数量递减」。

EffectDecoratorSystem.cs — 完整源码
public class EffectDecoratorSystem : AComponentSystem<AbilityEffect, EffectDecoratorComponent>,
    IAwake<AbilityEffect, EffectDecoratorComponent>
{
    public void Awake(AbilityEffect entity, EffectDecoratorComponent component)
    {
        if (entity.EffectConfig.Decorators != null)
        {
            foreach (var effectDecorator in entity.EffectConfig.Decorators)
            {
                if (effectDecorator is DamageReduceWithTargetCountDecorator reduceWithTargetCountDecorator)
                {
                    entity.AddComponent<EffectDamageReduceWithTargetCountComponent>();
                }
            }
        }
    }
}

EffectDamageReduceWithTargetCountComponent

装饰器组件 — 多目标伤害递减
public class EffectDamageReduceWithTargetCountComponent : EcsComponent
{
    public float ReducePercent { get; set; }  // 每多一个目标减少的百分比
    public float MinPercent { get; set; }     // 最低伤害百分比下限
}
装饰器模式的实际用途
MOBA 中常见场景：AOE 技能对第一个目标 100% 伤害，每多一个目标减 15%，最低 40%。

ReducePercent = 15，MinPercent = 40
• 1 目标：100% 伤害
• 2 目标：85% 伤害
• 3 目标：70% 伤害
• 5+ 目标：40% 伤害（触底）

装饰器不改变 Effect 本身的逻辑，而是在 DamageActionSystem 执行时检查是否存在装饰器组件，如果存在则修改最终伤害值。

5.12 Effect → Action 转换模式

Effect 系统中最重要的设计模式是 Effect → Action 的转换。Effect 是「能力声明」（静态配置），Action 是「行为执行」（运行时实例）。

Trigger 触发条件满足
如：技能命中目标、计时器到期
↓
创建 EffectAssignAction
携带 Target + AbilityEffect 引用
↓
AbilityEffectSystem.TriggerApply(effectAssign)
分发到对应的 EffectXxxSystem.TriggerApply
↓
OwnerEntity.XxxAbility.TryMakeAction()
通过工厂 Ability 创建具体 Action 实例
↓
XxxActionSystem.Execute(action)
执行实际效果（扣血/加血/加 Buff/...）

Effect → Action 映射表

Effect 类型组件工厂 AbilityAction 类型ActionSystem
DamageEffect EffectDamageComponent DamageAbility DamageAction DamageActionSystem
CureEffect EffectCureComponent CureAbility CureAction CureActionSystem
AddStatusEffect EffectAddBuffComponent AddStatusAbility AddBuffAction AddBuffActionSystem
AttributeModifyEffect EffectAttributeModifyComponent —（直接操作 Numeric） — —
ActionControlEffect EffectActionControlComponent —（直接通知 BuffSystem） — —
ShieldDefenseEffect EffectShieldDefenseComponent —（直接添加组件） — —
两种触发模式
上表揭示了两种不同的触发模式：

模式 A：通过 Action 执行（Damage/Cure/AddBuff）
Effect → TryMakeAction → Action → ActionSystem.Execute
适用于：一次性执行的行为，需要目标，需要记录战斗日志

模式 B：直接生效（Attribute/ActionControl/Shield）
TriggerApply 时直接操作组件或通知系统
适用于：持续性效果，通过 Enable/Disable 控制生命周期

Attribute 和 ActionControl 使用 Enable/Disable 机制（Buff 激活时生效，结束时回退）。它们不需要 Action 是因为效果是可逆的。

5.13 设计总结

组件即声明
AbilityEffect 挂什么组件就是什么效果。工厂方法 Create 根据配置类型自动挂载。配置驱动 + 组件组合 = 无限可扩展性。添加新效果类型只需：新配置类 + 新 Component + 新 System。
所有 System 结构统一
每个 EffectXxxSystem 都遵循相同模式：
• Awake：缓存配置引用
• Enable/Disable（可选）：持续效果
• TriggerApply（静态）：一次性效果
阅读任何一个就能理解全部。
TriggerApply 的手动分发
AbilityEffectSystem.TriggerApply 用 if (item is XxxComponent) 手动分发到各 System。这不是 EcsNode 的自动驱动，而是手动调度。好处是执行顺序可控；坏处是每加一种新 Effect 就得在这里多加一个 if。
已知局限
• TriggerApply 的 if 链不是开闭原则友好的
• 属性修改的 BaseValue 模式在 Disable 时没有回退逻辑（SetBase 是覆盖操作）
• 装饰器目前只实现了 DamageReduceWithTargetCount 一种，扩展点存在但不完善
